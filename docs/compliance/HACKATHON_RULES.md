Last verified: **July 18, 2026**

Authoritative sources:

- Official Rules: https://openai.devpost.com/rules
- Challenge requirements: https://openai.devpost.com/
- OpenAI Build Week page: https://openai.com/build-week/

This file is a working summary, not a replacement for the Official Rules. Recheck the Official Rules and challenge website immediately before submission. If sources conflict, the Official Rules and official challenge notices govern.

- Submission period: July 13, 2026 at 9:00 AM PT through **July 21, 2026 at 5:00 PM PT**.
- Lisbon deadline: **July 22, 2026 at 1:00 AM WEST**.
- Official Rules judging period: July 22 through August 5, 2026.
- Winners announced on or around August 12, 2026.
- The entry cannot ordinarily be changed after the submission deadline.

## Entrant and category

- Entry type: Team.
- Country: Portugal.
- Legal-age requirement: Confirmed by both team members.
- Authorized representative: Victoria.
- Selected category: **Apps for Your Life**.

The team must reconfirm that no member falls within an exclusion or conflict-of-interest category in the Official Rules.

## Required project

Flownee must:

- Be built with Codex and GPT-5.6.
- Fit the selected track.
- Install and run consistently on its intended platform.
- Function as shown in the demonstration and described in the submission.
- Be a working, non-trivial implementation.
- Remain accessible free of charge and without restriction to judges through the judging period.

Flownee is being created from scratch during the submission period. No pre-existing application, source code, prototype, design, or prompt baseline is being reused.

## Required submission materials

- A working project.
- The best-fitting category.
- A text description explaining features and functionality.
- A public YouTube demonstration video under three minutes.
- A code-repository URL for judging and testing.
- A README with setup, sample data where needed, and clear run instructions.
- A `/feedback` Codex Session ID for the project task where most core functionality was built.
- A working website, demo, or test build with testing instructions.

Flownee will use a public GitHub repository with an MIT License. No repository invitation should be necessary.

## Demo-video requirements

The video must:

- Be less than three minutes; judges need not watch beyond three minutes.
- Be publicly visible on YouTube.
- Clearly show Flownee working.
- Include audio explaining what was built and how Codex and GPT-5.6 were used.
- Exclude third-party trademarks, copyrighted music, or other protected material unless permission exists.

## README and Codex evidence

The final README must explain:

- Setup, execution, testing, and sample data where needed.
- How the team collaborated with Codex.
- Where Codex accelerated development.
- Which important product, engineering, and design decisions were made by humans.
- How GPT-5.6 and Codex contributed to the final result.

Preserve dated commits and Codex session records. Obtain the required `/feedback` Session ID after the task where most core functionality is built.

## Repository and testing

- The public repository must have relevant licensing.
- The Netlify deployment must be reachable without an account, payment, invitation, or user-supplied API key.
- Testing access must remain free and unrestricted through the judging period.
- All required materials must be in English or include English translations.
- Judges may evaluate only the description, images, and video, so those artifacts must communicate the complete value even if the app is not executed.

## Intellectual property and integrations

- The submission must be original, solely owned by the entrant team, and must not infringe another party's rights.
- Open-source licenses must be followed, and the project must enhance or build on underlying open-source functionality.
- The team must be authorized to use every third-party SDK, API, dataset, font, icon, image, audio asset, and other integration under its applicable terms.
- The submission and repository must contain no malicious code.
- The project must not have prohibited financial or preferential support from OpenAI or Devpost as described in the Official Rules.

## Evaluation

Stage One is pass/fail:

- Reasonable fit with the challenge theme.
- Reasonable application of the required technologies.

Stage Two uses four equally weighted criteria:

| Criterion | Weight | Flownee evidence target |
|---|---:|---|
| Technological Implementation | 25% | Working non-trivial PWA, substantive Codex use, GPT-5.6 reasoning, reliable structured outputs, and testing |
| Design | 25% | Complete voice-to-action journey, coherent responsive UX, and resilient interaction states |
| Potential Impact | 25% | Specific busy-adult audience, early evidence, and a demonstrated reduction in capture and decision friction |
| Quality of the Idea | 25% | Voice-first incremental inbox plus continuously updated, explainable next-action guidance |

Technological Implementation is the first tie-break criterion.

## High-risk disqualification conditions

- Late, incomplete, or materially altered post-deadline submission.
- Ineligible participant, representative, or conflict of interest.
- Failure to use Codex and GPT-5.6 meaningfully.
- Project does not fit an eligible track or cannot run as represented.
- Missing or inaccessible repository, deployment, video, or testing path.
- Missing Codex `/feedback` Session ID.
- Unlicensed or unauthorized code, data, integrations, trademarks, or media.
- Non-English material without required English translation.
- False statements about functionality, ownership, research, or AI use.
- Malicious code or prohibited sponsor/administrator support.
